# Implementation Phase Workflow

## Purpose

This document mirrors the template's implementation-phase workflow for coding work performed in the repository.

---

## Core Steps

1. Set up a clear repository structure for the implementation.
2. Follow established coding and documentation standards.
3. Deliver work incrementally in small, reviewable changes.
4. Add tests where appropriate for scripts or software components.
5. Update project context and usage documentation as implementation evolves.

---

## Expectations

- Keep code maintainable and well-scoped.
- Prefer reusable patterns over one-off implementations.
- Document assumptions, dependencies, and operational constraints.

---

## MTSA Note

Use this workflow mainly for scripts, tooling, and automation in `scripts/`, `tools/`, or future implementation modules.
