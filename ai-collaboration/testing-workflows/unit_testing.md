# Unit Testing Workflow

## Purpose

Reference workflow for testing individual functions, helpers, and modules in isolation.

---

## Core Practices

- Keep tests fast and isolated.
- Cover normal cases, edge cases, and failure cases.
- Mock external dependencies where appropriate.
- Keep test names explicit and behavior-focused.

---

## MTSA Note

Apply this mainly to Python or shell helpers, validation tooling, and automation scripts in this repo.
