# Regression Testing Workflow

## Purpose

Reference workflow for ensuring previously working behavior still works after changes.

---

## Core Practices

- Preserve tests for critical paths and fixed bugs.
- Run smoke tests frequently and deeper suites before larger releases.
- Track flaky or obsolete tests and clean them up promptly.
- Prioritize tests that protect the highest-risk workflows.

---

## MTSA Note

Use this for automation, scripts, templates, and repeatable document-generation workflows where changes could break prior behavior.
