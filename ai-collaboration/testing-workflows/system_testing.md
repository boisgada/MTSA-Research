# System Testing Workflow

## Purpose

Reference workflow for validating complete end-to-end behavior in a production-like environment.

---

## Core Practices

- Test complete user or operator workflows.
- Use production-like configuration where feasible.
- Validate critical paths, error handling, and recovery behavior.
- Track results and unresolved defects clearly.

---

## MTSA Note

For MTSA, this is most relevant when validating larger automation pipelines or tool-assisted workflows rather than document edits alone.
