# Quick Start Guide

## Get Started in 5 Minutes

### Step 1: Review Project Context (1 minute)

Review `ai-collaboration/project-context.md` to understand the MTSA compliance project:
- Project overview and objectives
- Key research findings
- Regulatory requirements and deadlines
- Current project status

### Step 2: Review Rules of Engagement (1 minute)

Open `ai-collaboration/rules-of-engagement.md`:
- Understand collaboration guidelines
- Review verified references requirement
- Note session workflow patterns

### Step 3: (Optional) Set Up Queue System or Memory Bank (1-2 minutes)

**If using queue system**:
- Review `ai-collaboration/queue-system-guide.md`
- Customize `queue/master-queue.md` as needed

**If project is complex**:
- Review `ai-collaboration/memory-bank/README.md`
- Decide if memory-bank structure is right for your project

### Step 4: Start Your First Session (1 minute)

1. Open your project in Cursor/VS Code
2. Start a new AI chat session
3. Say: "Please read `ai-collaboration/project-context.md` and `ai-collaboration/rules-of-engagement.md` to understand this project"
4. If using queue system, mention: "Review `queue/master-queue.md` for current work items"
5. Provide your session objectives

---

## What's Next?

### During Your Session

- Reference existing documents when asking questions
- Build on previous work
- Follow the session workflow in `rules-of-engagement.md`

### After Your Session

- Document outcomes using `ai-collaboration/session-notes-template.md`
- Update `project-context.md` with new decisions or findings
- Note any follow-up items

### Ongoing

- Keep `project-context.md` updated as your project evolves
- Use session notes to track progress
- Leverage awesome-copilot resources for specialized tasks

## Maintenance

### Keeping Submodules Up-to-Date

If your project uses git submodules (like `awesome-copilot`):

1. **Check status**: `git submodule status`
2. **Update all**: `git submodule update --remote`
3. **Or use helper script**: `./scripts/update-submodules.sh`

See `documentation/submodule-maintenance.md` for detailed guide (if available).
