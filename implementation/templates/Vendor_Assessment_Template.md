# Vendor Security Assessment Template

## Purpose

This template provides a structure for assessing vendor systems that may be critical assets under MTSA. Use for evaluating vendor security posture and MTSA compliance capabilities.

---

## Assessment Structure

### 1. Vendor Information
- Vendor name
- Vendor contact information
- Vendor system/service description
- Contract information

### 2. System Information
- System name and description
- System function
- System criticality assessment
- System dependencies

### 3. Security Assessment
- Security policies and procedures
- Security controls implementation
- Access controls
- Encryption and data protection
- Monitoring and detection
- Incident response capabilities

### 4. Compliance Assessment
- MTSA awareness
- MTSA compliance capabilities
- Compliance certifications (ISO 27001, SOC 2, etc.)
- Compliance reporting capabilities
- Audit capabilities

### 5. Risk Assessment
- Security risks identified
- Compliance risks identified
- Operational risks identified
- Risk prioritization
- Risk mitigation strategies

### 6. Assessment Results
- Overall assessment rating
- Key findings
- Strengths identified
- Areas for improvement
- Recommendations

### 7. Decision
- Assessment decision (Approve/Approve with Conditions/Reject)
- Conditions (if applicable)
- Follow-up requirements
- Next assessment date

### 8. Appendices
- Supporting documentation
- Vendor responses
- Additional information

---

## Template Usage

**Assessment:**
- Complete for all critical vendor systems
- Use standardized criteria
- Document all findings
- Maintain assessment records

**Decision:**
- Make decision based on assessment
- Document rationale
- Set conditions if needed
- Plan follow-up
