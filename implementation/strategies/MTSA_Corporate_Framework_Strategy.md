# MTSA Compliance Strategy: Leveraging Corporate Framework for Multi-Site Efficiency

## Executive Summary

This strategy document provides a **corporate-level approach** to MTSA compliance that leverages your existing mature cybersecurity program (IT/OT), consolidated corporate requirements framework, and best-in-class OT architecture to **minimize duplication** across your four MTSA-regulated sites.

**Key Principle**: Document corporate controls once, reference in site-specific plans, add only site-specific elements.

**Goal**: Achieve MTSA compliance for 4 sites with minimal duplication, leveraging existing corporate cybersecurity investments.

## 1. Strategic Approach Overview

### 1.1 Core Strategy: "Document Once, Reference Many"

**Approach**:
1. **Corporate Level**: Document corporate cybersecurity framework, controls, and processes
2. **Site Level**: Create site-specific Cybersecurity Plans that reference corporate framework
3. **Site-Specific Additions**: Add only maritime-specific and site-unique elements

### 1.2 Benefits

- **Minimize Duplication**: Document corporate controls once, reference in all 4 site plans
- **Leverage Existing Work**: Use existing corporate cybersecurity framework
- **Consistency**: Ensure consistent security approach across sites
- **Efficiency**: Reduce effort by 70-80% compared to site-by-site approach
- **Maintainability**: Update corporate framework once, all sites benefit

### 1.3 Key Assumptions

- Corporate cybersecurity framework already addresses most MTSA technical requirements
- Corporate training program can be extended for MTSA
- Corporate incident response can be extended for NRC reporting
- Corporate risk assessment methodology can be extended for maritime operations
- Site-specific work focuses on maritime operations and integration with FSP

## 4. Site-Specific Plan Structure

### 4.1 Site Cybersecurity Plan Template

Each site plan should follow this structure:

#### Section 1: Site Overview
- Site description
- Site operations
- Maritime operations description
- Site personnel

#### Section 2: Corporate Framework Reference
- **Reference to Corporate Framework**: "This site implements the corporate cybersecurity framework documented in [Corporate Framework Document]"
- **Corporate Controls**: Reference specific corporate controls that satisfy MTSA requirements
- **Corporate Capabilities**: Reference corporate capabilities (training, incident response, etc.)

#### Section 3: Site-Specific Maritime Operations
- Maritime operations description
- Critical asset identification (maritime-focused)
- TSI pathway analysis
- Maritime threat assessment

#### Section 4: Site-Specific Extensions
- Maritime-specific training content
- NRC reporting procedures (extension of corporate incident response)
- Maritime-specific drill/exercise scenarios
- Site-specific contacts and coordination

#### Section 5: FSP Integration
- Integration with Facility Security Plan
- Coordination between cyber and physical security
- Unified security approach

#### Section 6: Site-Specific Assessment
- Site-specific risk assessment (using corporate methodology)
- Site-specific vulnerability assessment
- Site-specific control gaps (if any)

### 4.2 Site Plan Benefits

1. **Minimal Duplication**: Mostly references corporate framework
2. **Site-Specific Focus**: Focuses on maritime operations and site-specific elements
3. **Efficient Development**: Can be developed quickly using template
4. **Consistent Structure**: All sites follow same structure
5. **Easy Maintenance**: Update site-specific elements only

## 5. Requirement-by-Requirement Strategy

### 5.1 Requirement 1: Cybersecurity Plan

#### Corporate Contribution
- **Corporate Framework Document**: Comprehensive corporate cybersecurity framework
- **Corporate Policies**: Corporate security policies and standards
- **Corporate Procedures**: Corporate security procedures

#### Site-Specific Additions
- Site-specific maritime operations description
- Site-specific critical asset identification
- Site-specific TSI pathway analysis
- Site-specific FSP integration
- Site-specific contacts and coordination

#### Implementation Approach
1. **Develop Corporate Framework Document** (once, for all sites)
2. **Create Site Plan Template** (once, reuse for all sites)
3. **Develop Site-Specific Plans** (4 plans, minimal effort - mostly fill-in template)
4. **Submit Site Plans to Coast Guard** (4 submissions)

**Effort Reduction**: ~75% compared to developing 4 independent plans

### 5.2 Requirement 2: Cybersecurity Officer (CySO)

#### Corporate Contribution
- **Corporate Information Security Manager**: Can serve as corporate CySO coordinator
- **Corporate Cybersecurity Team**: Supports site CySOs

#### Site-Specific Additions
- Site-specific CySO designation (can be same person for multiple sites if appropriate)
- Site-specific CySO responsibilities (maritime-focused)
- Site-specific contact information

#### Implementation Approach
1. **Designate Corporate CySO Coordinator** (once)
2. **Designate Site CySOs** (4 designations, can leverage corporate team)
3. **Document Site-Specific Responsibilities** (minimal - mostly maritime-specific)

**Effort Reduction**: ~60% compared to independent site CySOs

### 5.3 Requirement 3: Cybersecurity Training

#### Corporate Contribution
- **Corporate Training Program**: Existing corporate cybersecurity training
- **Corporate Training Framework**: Training delivery and recordkeeping

3. **Maintain Site-Specific Records** (4 record sets, using corporate system)

**Effort Reduction**: ~80% compared to developing site-specific training programs

## 6. Implementation Model

### 6.1 Corporate-Level Work (Do Once)

#### Phase 1: Corporate Framework Documentation (Months 1-3)
1. Document corporate cybersecurity framework
2. Document corporate technical controls
3. Document corporate organizational capabilities
4. Document OT architecture and controls
5. Create corporate framework document

**Effort**: ~3-4 months, 1-2 FTE

#### Phase 2: Corporate Framework Extensions (Months 4-6)
1. Develop maritime-specific training modules
2. Develop NRC reporting procedures
3. Develop maritime assessment methodology
4. Develop maritime drill/exercise scenarios
5. Extend corporate incident response for NRC

**Effort**: ~2-3 months, 1 FTE

#### Phase 3: Site Plan Template Development (Month 7)
1. Create site plan template
2. Create site assessment template
3. Create site training template
4. Create site drill/exercise template

**Effort**: ~1 month, 0.5 FTE

**Total Corporate Effort**: ~6-8 months, 1-2 FTE

### 6.2 Site-Level Work (Do Per Site)

#### Phase 4: Site-Specific Development (Months 8-10)
For each of 4 sites:
1. Conduct site-specific maritime operations assessment
2. Identify site-specific critical assets
3. Conduct site-specific TSI pathway analysis
4. Develop site-specific plan using template
5. Integrate with site FSP

**Effort Per Site**: ~2-3 weeks, 0.5 FTE
**Total Site Effort**: ~2-3 months, 2 FTE (can be parallel)

### 6.3 Total Effort Comparison

#### Independent Site-by-Site Approach
- **Per Site**: ~6-8 months, 1-2 FTE
- **4 Sites**: ~24-32 months, 4-8 FTE
- **Total Effort**: ~24-32 person-months

#### Corporate Framework Approach
- **Corporate Work**: ~6-8 months, 1-2 FTE
- **Site Work**: ~2-3 months, 2 FTE
- **Total Effort**: ~8-11 person-months

**Effort Reduction**: ~65-70% reduction in total effort

## 7. Cost and Effort Optimization

### 7.1 Cost Optimization Strategies

#### 1. Leverage Existing Corporate Resources
- Use existing corporate cybersecurity team
- Use existing corporate training resources
- Use existing corporate tools and systems
- Use existing corporate processes

#### 2. Minimize Site-Specific Work
- Focus site work on maritime-specific elements only
- Use templates for site plans
- Reuse corporate documentation
- Standardize site-specific elements

#### 3. Parallel Site Work
- Conduct site assessments in parallel
- Develop site plans in parallel
- Train site personnel in parallel
- Conduct site drills/exercises in parallel

#### 4. Reuse Across Sites
- Maritime training modules (develop once, use at all sites)
- Maritime scenarios (develop once, use at all sites)
- NRC reporting procedures (develop once, use at all sites)
- Assessment methodology (develop once, use at all sites)

### 7.2 Effort Optimization Strategies

#### 1. Corporate Framework First
- Complete corporate framework documentation first
- Use corporate framework as foundation for all sites
- Ensure corporate framework is comprehensive

#### 2. Template-Based Approach
- Create templates for all site-specific documents
- Use templates to ensure consistency
- Minimize site-specific development effort

#### 3. Incremental Implementation
- Start with highest-priority sites
- Learn from first site implementation
- Apply lessons learned to subsequent sites

#### 4. Automation Where Possible
- Use corporate systems for recordkeeping
- Use corporate systems for training delivery
- Use corporate systems for incident reporting
- Automate documentation where possible
