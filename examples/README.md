# Example Files

## Purpose

This folder contains example files to help you understand how to customize the template for different types of projects.

---

## Contents

### Project Context Examples

- **project-context-compliance-example.md**: Example for compliance/regulatory projects
  - Based on patterns from successful compliance projects
  - Includes verified references requirement
  - Shows regulatory deadline tracking
  - Demonstrates framework leverage approach

- **project-context-research-example.md**: Example for research projects
  - Shows research methodology documentation
  - Demonstrates research question tracking
  - Includes synthesis and deliverables sections
  - Suitable for exploratory research

---

## How to Use These Examples

1. **Review the Examples**: Read through the examples to understand different approaches
2. **Identify Your Project Type**: Determine which example is closest to your project
3. **Customize**: Use the example as a starting point, but customize for your specific needs
4. **Remove Placeholders**: Replace all `[placeholder]` text with your actual information
5. **Add/Remove Sections**: Add sections specific to your project, remove sections that don't apply

---

## Creating Your Own Project Context

1. Start with the template: `../ai-collaboration/project-context.md`
2. Review relevant examples in this folder
3. Customize sections for your project
4. Remove sections that don't apply
5. Add project-specific sections as needed
6. Keep it updated as your project evolves

---

## Notes

- These examples are **references**, not templates to copy directly
- Extract patterns and structures, not specific content
- Adapt the examples to match your project's needs
- Remember to keep your project context generic if it will be in a public repository
