# MTSA Glossary (Common Terms)

## Purpose

Provide consistent terminology across MTSA documents. Use this glossary when drafting or reviewing content.

---

## Terms

- **MTSA**: Maritime Transportation Security Act requirements for regulated facilities.
- **FSP**: Facility Security Plan. The physical security plan to integrate with the Cybersecurity Plan.
- **Cybersecurity Plan**: MTSA-required plan covering prevention, detection, response, and recovery; integrates with FSP.
- **CySO**: Cybersecurity Officer responsible for MTSA plan development, maintenance, and coordination.
- **TSI (Transportation Security Incident)**: Security incident that results in significant disruption to transportation systems.
- **NRC**: National Response Center (24/7). Immediate reporting point for MTSA cyber incidents.
- **Coast Guard Submission**: Cybersecurity Plan and evidence package submitted to USCG for approval.
- **Critical Assets**: Assets whose compromise could lead to a TSI or significant operational impact.
- **OT / ICS**: Operational Technology / Industrial Control Systems (e.g., PLCs, SCADA, DCS).
- **Zones / Conduits**: Segmentation model (per 62443) grouping assets (zones) and the controlled paths between them (conduits).
- **SL-T (Target Security Level)**: Desired security level for a zone (IEC 62443), reflecting required protections.
- **AAR**: After Action Report documenting drill/exercise findings and improvements.
- **SoA (Statement of Applicability)**: ISO 27001 document listing applicable controls and justifications.
- **Compensating Control**: Temporary or alternative control used when a primary control (e.g., patch) cannot be applied promptly.
- **Drill vs. Exercise**: Drills test specific procedures; exercises are broader, integrating multiple functions/teams.
- **Evidence Pack**: Curated artifacts (plans, logs, AARs, diagrams, access reviews) prepared for regulators/auditors.
- **OT Monitoring**: Logging and detection tuned for ICS protocols and assets; often separate or tiered from IT monitoring.
- **Vendor Remote Access (VRA)**: Controlled, time-bound access for vendors into OT; should include approvals, monitoring, and strong authentication.

---

## Usage Notes

- Prefer these terms in headers and summaries; add local definitions sparingly.
- When adopting new framework terminology (e.g., CSF v2.0), dual-label where helpful: `GV.RR-01 (formerly ID.GV-1)`.
- Keep this glossary updated when new technologies, roles, or regulatory terms enter scope.
